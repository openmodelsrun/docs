# Version 1.0 Roadmap

This page tracks progress toward the OpenModels v1.0 release. Each section represents a milestone with individual tasks showing current status.

**Target:** Q3 2026

**Status Legend:**
- **Done** — completed and shipped
- **In Progress** — actively being worked on
- **Planned** — scheduled, not yet started

---

## Phase 1: Registry Foundation — Done

Establish the public registry as the canonical source of truth for models, providers, and mappings.

- [x] Define JSON Schema for models, providers, and mappings
- [x] Create validation script with schema, duplicate, and referential integrity checks
- [x] Set up GitHub Actions CI for automated PR validation
- [x] Populate initial model definitions (60+ models across 15+ families)
- [x] Populate provider definitions (30+ providers with verified endpoints)
- [x] Add provider-model mappings with pricing data (80+ mappings)
- [x] Verify pricing against official provider documentation
- [x] Support extended pricing dimensions (cache, image, audio, reasoning)

**Status:** Done

---

## Phase 2: API — Done

Build the REST API for model discovery, comparison, and provider lookup.

- [x] `GET /api/models` — list and search models with sort options
- [x] `GET /api/models/:id` — model details
- [x] `GET /api/models/:id/providers` — providers for a model
- [x] `GET /api/models/:id/compare` — compare providers
- [x] `GET /api/models/popular` — relevance-ranked popular models
- [x] `GET /api/providers` — list providers
- [x] `GET /api/providers/:id` — provider details
- [x] `GET /api/stats` — registry statistics (counts + last sync)
- [x] `GET /api/search` — unified full-text search
- [x] `GET /api/search/index` — lightweight client-side search index
- [x] `GET /api/health` — system health check
- [x] Caching layer with TTL-based invalidation
- [x] Pagination and filtering across all list endpoints
- [ ] Rate limiting and request throttling
- [ ] OpenAPI 3.1 spec generation with Swagger UI

**Status:** Done (rate limiting and OpenAPI docs planned for v1.0)

---

## Phase 3: Telemetry — Done

Real-time monitoring of provider health, latency, and availability.

- [x] Health probe scheduler (5-minute intervals)
- [x] Latency probe scheduler (15-minute intervals)
- [x] Time-to-first-token measurement
- [x] Total response time measurement
- [x] `GET /api/telemetry/health/:provider_id` endpoint
- [x] `GET /api/telemetry/latency/:provider_id` endpoint
- [x] `GET /api/telemetry/ranked/:model_id` — ranked providers by performance
- [x] Historical telemetry data retention (30 days)
- [x] "Unknown" status for providers with no/stale health data
- [x] Health probe accepts 401/403 as reachable (no false negatives)
- [ ] Alerting for provider outages

**Status:** Done (alerting planned for v1.0)

---

## Phase 4: Web Interface — Done

Web application for browsing, searching, and comparing models at [openmodels.run](https://openmodels.run).

- [x] Model search and browse page with sort options
- [x] Model detail page with provider list and error handling
- [x] Provider comparison view (pricing, latency, uptime)
- [x] Command Palette (`Cmd+K` / `Ctrl+K`) for instant global search
- [x] Popular Models section with relevance scoring
- [x] Provider logo system with SVG assets and letter fallback
- [x] Provider Marquee (auto-scrolling logo strip)
- [x] Ecosystem Graph (interactive node visualization)
- [x] Recently Added section with relative timestamps
- [x] Category-based navigation (capability, modality, license)
- [x] Dark/light mode theming
- [x] Responsive layout for mobile devices
- [ ] SEO optimization (meta tags, sitemap, structured data)

**Status:** Done (SEO optimization planned for v1.0)

---

## Phase 5: Documentation — Done

Comprehensive documentation site for users and contributors.

- [x] Docs site setup with Nextra
- [x] Architecture documentation (overview, data flow, schemas)
- [x] Contributing guides (adding models, adding providers)
- [x] Roadmap page
- [x] Getting Started section (quickstart, concepts, installation)
- [x] API Reference section with endpoint documentation
- [x] SEO and sitemap configuration
- [ ] Deployment to production

**Status:** Done (deployment pending)

---

## Phase 6: Production Hardening — Planned

Final polish and production readiness for v1.0.

- [ ] API rate limiting (100 req/min per IP)
- [ ] OpenAPI 3.1 spec auto-generation with Swagger UI
- [ ] Provider outage alerting (webhook + email)
- [ ] SEO optimization (structured data, meta tags, sitemap)
- [ ] Lighthouse scores > 90 (Performance, Accessibility, SEO)
- [ ] Security audit (no critical or high findings)
- [ ] Load testing (100 req/s sustained)
- [ ] Production deployment with zero-downtime updates

**Status:** Planned

---

## Release Criteria

Before tagging v1.0, the following must be complete:

- [ ] All Phase 1–6 tasks marked as done
- [ ] API uptime > 99.5% over 30-day window
- [ ] Documentation covers all public endpoints
- [ ] Lighthouse scores > 90 (Performance, Accessibility, SEO)
- [ ] Security audit passed (no critical or high findings)
- [ ] Load testing confirms API handles 100 req/s sustained
